# Benefit Description

Kimi Code is a service within the Kimi Membership Plan that provides programming-related support and features, enabling members to complete personal development tasks more efficiently.

Please note that this benefit is for personal development only and may not be used in enterprise scenarios. **Users must confine programming-related usage to Kimi CLI and any other coding agents we permit or listed and updated on the official Kimi website**. For enterprise requirements, please consult the Moonshot AI Open Platform, which supports prepaid, usage-based billing, model selection, and broader integration scenarios.

## Billing Method

### Billing Logic

**Kimi Code** quota are included as part of your Kimi Membership benefits; no additional fees are required. You can utilize your credits through the following methods:
- **API Key Integration**: When using an API Key generated via the [Kimi Code Console](https://www.kimi.com/code/console) on supported platforms, requests will be deducted from your plan's quota, and your available credits will refresh according to your billing cycle.
- **Direct Account Login**: By logging into your Kimi account within Kimi Code for CLI or Kimi Code for VS Code, your Kimi Code usage will be automatically applied to and deducted from your active subscription plan.

### Quota Refresh Logic

Starting from the subscription activation date (D1), request quotas are refreshed on a 7-day rolling cycle:

- Upon activation, the first quota period starts immediately and remains valid from **D1 to D7 (inclusive)**.
- On **D8**, the quota automatically refreshes and remains valid from **D8 to D14**.
- The same pattern continues thereafter.

Unused quotas do **not** carry over to the next cycle.

## Scope of Use

The **Kimi Code** benefit is primarily intended for use with **Kimi CLI**. Recognising that Kimi CLI is still undergoing continuous optimisation, and to better support developer workflows, members may also configure and use the API Key provided under this benefit within compatible third-party environments where technically feasible. **Requests made through these platforms will likewise consume the applicable quota.** Additional compatible coding agents may be supported in the future.
