# Kimi Code 使用规范

Kimi Code 致力于为全球开发者提供智能、高效的 AI 编程服务，用模型能力的持续跃迁、Agent 产品力的极致打磨、Token Plan 的范式革新，缩短人类想象到现实的路径，让通用 Agent 普惠每个有想法的个体。为保证每位 Kimi Code 用户拥有更好的体验，我们提出以下使用规范。


## 使用边界

Kimi Code 订阅仅限编程工具交互式使用（Kimi CLI、VS Code、Claude Code、OpenCode、OpenClaw 等），套餐 API Key 不得擅自用于自建应用、SaaS 产品、服务端中转或任何非本地编码工具场景。如需此类用途或企业级集成，请前往 [Kimi 开放平台](https://www.moonshot.ai/) 或者直接联系 [code@moonshot.ai](mailto:code@moonshot.ai)，会有专业销售为您服务。


## 🚫 平台红线

为了保障 Kimi Code 会员用户能够享受到更加稳定的模型服务，我们禁止以下行为：

- 禁止以商业化为目的的账号转售、API Key 售卖或未经许可私自向第三方提供访问权限
- 禁止以任何形式汇集、转发或二次分售 Kimi Code 推理能力
- 禁止伪造或篡改客户端身份信息
- 禁止进行模型蒸馏训练等行为

如您违反了以上规则，我们会按情节进行限流或账号封禁等处置。

:::info 🤖 误判申诉
若你被错误限流或封禁，请到 [Kimi 官网](https://www.kimi.com/) 通过点击头像 → 用户反馈进行申诉，并提交账号 ID、封禁时间、使用场景说明及证明截图。若核实为误判，我们将立即解除限制；因误封造成的不便，我们会酌情赠送额外额度以示歉意。
:::

## 💬 常见问题

**Q：我是重度用户，每天高强度使用，会被误封吗？**

**A：** 不会。风控识别会考虑多种特征，对于 IP 地址、对话状态等使用指标正常的重度用户通常不会触发处置。

**Q：我用 VPN 或公司代理访问，会被误封吗？**

**A：** 通过代理访问 Kimi（正向代理）没问题；把自己的账号给别人转发（反向代理），属于违规。
