# Kimi Datasource

kimi-datasource 是 Kimi Code 官方数据插件，通过自然语言直接查询金融行情、宏观经济、企业工商和学术文献，无需手动调用接口或申请数据账号。

**前提**：需通过 `/login` 以 Kimi Code 账号完成 OAuth 登录，插件依赖本地凭据访问数据服务。

## 安装

**从 marketplace 安装（推荐）**

```
/plugins marketplace
```

找到 Kimi Datasource，按 `Space` 安装。

**直接安装**

```
/plugins install https://cdn.kimi.com/kimi-code/plugins/official/kimi-datasource.zip
```

安装后开启新会话使插件生效：

```
/new
```

验证安装成功（MCP server `data` 状态应为 connected）：

```
/plugins info kimi-datasource
```

## 数据源

插件接入 6 个数据源，Agent 会根据问题自动选择：

| 数据源 | 说明 |
| --- | --- |
| `stock_finance_data` | A 股、港股、美股行情与财务数据 |
| `yahoo_finance` | 全球主要金融市场数据 |
| `world_bank_open_data` | 世界银行宏观经济数据 |
| `tianyancha` | 中国企业工商、股权与司法信息（仅大陆企业） |
| `arxiv` | 学术预印本平台 |
| `scholar` | 学术文献检索 |

::: tip 使用建议
- A 股股票代码需带市场后缀，如 `600519.SH`、`0700.HK`
- 企业查询必须使用全称，如"北京字节跳动科技有限公司"
- 部分 API 返回完整数据时会写入本地 CSV 文件，需在 prompt 中指定 `file_path`（如 `/tmp/result.csv`）
:::

## 使用示例

直接用自然语言描述需求即可，无需指定数据源。

**A 股财务对比**

```
帮我把宁德时代和比亚迪 2024 年的资产负债率、净利润率、ROE 拉出来，
做个对比表，判断哪家财务结构更稳健。
```

**宏观经济研究**

```
拉一下中美日三国过去 10 年的 GDP 增速和 CPI 数据，
做成年度对比表，标出关键政策节点的影响。
```

**企业背景调研**

```
我要和一家公司签合同，帮我查"XX 科技有限公司"的工商注册、
股权结构、司法纠纷和失信被执行记录，给一个风险评估。
```

**学术文献调研**

```
帮我找一下近三年关于大语言模型代码生成能力的综述论文，
整理成包含标题、作者、核心结论的表格。
```

## 深度案例：多数据源联合调研

> 以下展示了 Kimi Code CLI 特有的**多步骤数据流水线**能力——普通 AI 对话无法完成这类任务，因为它需要跨工具的文件传递、本地代码执行和长上下文状态保持。

**场景**：产品团队在进入新行业前，需要对目标市场、潜在合作伙伴和技术路线同时做一次系统性调研，结果整理成可直接使用的报告。

```
帮我对"智能仓储"这个方向做一次系统调研，请按以下步骤依次完成，
每步完成后继续下一步，不要停下来等我确认。

【第一步 — 宏观市场背景】
从世界银行数据拉取中国过去 10 年的制造业增加值占 GDP 比重、
工业机器人密度（每万名工人机器人数量），保存到 /tmp/macro_data.csv。
打印前 5 行确认。

【第二步 — 头部企业调研】
用企业数据查询以下 3 家公司的工商注册信息、主要股东、
近 3 年融资情况和司法风险记录（逐一查询）：
- 海康威视
- 旷视科技
- 极智嘉

每家公司的结果保存到 /tmp/<公司名>_profile.txt。

【第三步 — 技术路线调研】
从学术数据库搜索 2023 年以来关于"仓储机器人路径规划"和
"多 AGV 调度算法"的高引用论文，各取前 5 篇，
整理标题、作者、核心方法到 /tmp/papers.csv。

【第四步 — 综合分析报告】
读取前三步生成的所有文件，撰写一份综合调研报告，写入 /tmp/report.md：

  # 智能仓储行业调研报告
  ## 一、宏观背景（附关键数据图表文字描述）
  ## 二、主要玩家分析（含风险提示）
  ## 三、核心技术路线（含文献支持）
  ## 四、进入建议与潜在合作伙伴推荐

报告完成后打印文件路径和字数。
```

**为什么这个任务需要 CLI：**

| 能力 | 普通 AI 对话 | Kimi Code CLI |
| --- | --- | --- |
| 查询真实企业工商数据 | ✗ 无法访问实时数据库 | ✓ 通过 kimi-datasource MCP 调用 |
| 多数据源联合分析 | ✗ 每次需要单独操作 | ✓ 单次会话完成全部步骤 |
| 结果写入本地文件 | ✗ 对话结束即消失 | ✓ 写入 `/tmp/*.csv / *.md` 永久保留 |
| 长流程不丢上下文 | ✗ 步骤多时容易遗忘 | ✓ 文件系统作为外部记忆 |

## 注意事项

- 数据查询按次计费，消耗 Kimi Code 账号额度
- 插件为只读查询，不提供任何写入或交易功能
- AI 输出内容仅供参考，不构成任何商业决策建议
